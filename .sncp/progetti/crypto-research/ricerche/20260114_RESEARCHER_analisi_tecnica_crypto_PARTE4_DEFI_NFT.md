## DEFI/NFT TECHNICAL CHALLENGES

### DeFi Complexity üß©

**LP Token Valuation**:
```
Uniswap V3 LP position:
- Non-fungible (NFT-based)
- Range orders (concentrated liquidity)
- Impermanent loss variable
- Fee earnings accumulated

How to value in real-time?
Current price + unclaimed fees + IL calculation
```

**Cross-Chain Positions**:
```
Same asset on multiple chains:
- ETH on Ethereum
- Bridged ETH on Polygon (WETH)
- Staked ETH on Lido (stETH)
- Arbitrum ETH

All same or different? How aggregate?
```

**Yield Farming Chains**:
```
User deposits USDC ‚Üí Aave (lending)
Aave gives aUSDC ‚Üí Stake in Curve
Curve gives CRV rewards ‚Üí Lock for veCRV
veCRV gives boost ‚Üí More CRV rewards

5 layers deep. How track P&L?
```

### NFT Valuation Challenges üìä

**Floor Price ‚â† Real Value**:
```
Bored Ape #1234:
- Floor price: 10 ETH
- Has rare traits: +50%?
- Has blue background: +20%?
- Market illiquid: -30%?

Real sellable price? ???
```

**Rarity Calculation**:
```
NFT with:
- Gold fur (5% rarity)
- Laser eyes (2% rarity)
- Crown (1% rarity)

Total rarity = 5% * 2% * 1% = 0.0001%?
Or additive: 5% + 2% + 1% = 8%?
Or something else?

Every tool uses different formula!
```

**Cross-Marketplace Liquidity**:
```
Same NFT listed on:
- OpenSea: 12 ETH
- Blur: 11.5 ETH
- LooksRare: 11.8 ETH

Which is "true" price?
Best bid? Lowest ask? Volume-weighted?
```

### On-Chain Data Indexing

**The Graph** (most common):
```javascript
// Subgraph query example
{
  user(id: "0x123...") {
    liquidityPositions {
      pair {
        token0 { symbol }
        token1 { symbol }
      }
      liquidityTokenBalance
    }
  }
}
```

**Benefits**:
- Decentralized indexing
- Protocol-specific subgraphs
- GraphQL queries

**Limitations**:
- Latency (minutes behind)
- Coverage gaps (new protocols)
- Complex to deploy

### Direct Blockchain Reads

**Alchemy/Infura RPC**:
```javascript
// Read LP balance directly
const balance = await contract.balanceOf(userAddress);
const totalSupply = await contract.totalSupply();
const reserves = await contract.getReserves();

// Calculate share
const userShare = (balance / totalSupply) * reserves;
```

**Benefits**:
- Real-time
- No middleware
- Full control

**Limitations**:
- Complex calculations
- Must know contract ABIs
- Rate limits

### Protocol APIs (where available)

**Aave API**:
```
GET /users/{address}/summary

```
